Installation and Usage

Tina is currently being developed and tested on Linux/i386.

It should be very simple to install and use Tina:

  1. Unpack
    1. Choose, and cd into, a convenient directory for the package.
    2. type "tar -xvzf tina-0.1.0.tar.gz" to unpack the package.
    3. Read the documentation. Really.
  2. Build
    1. If you don't have gcc installed, adjust the definitions for CC near the top of src/Makefile.
    2. (optional) Change the default config by setting the #define in src/config.h.
    3. (optional) Add your own mime type to the end of mime_types.txt if you really know what you at doing.
    4. type "make". Report any errors to me for resolution, or strike out on your own.
  3. Install
    • If build success,then type "sudo make install" to install Tina.
  4. Configure
    1. The config file for Tina is "/etc/tina/tina.conf".
    2. Edit "/etc/tina/tina.conf" according to your choices . Read through this file to see what other features you can configure.
    3. The default root ducument for tina is "/var/tina/www",so if you don't modify the config,you must create this directory by your self.just type "sudo mkdir -p /var/tina/www" to create it.
    4. The default index page for tina is "index.html" which you can find it in "conf/",if you don't modify the default config,you should copy this file or any other file named "index.html" into root document which is "/var/tina/www" by type "sudo cp conf/index.html /var/tina/www".because the "index.html" needs "logo.png",so you should copy the "logo.png" which is in "conf/" into the "/var/tina/www" by type "sudo cp conf/logo.png /var/tina/www" also.
    5. (optional)There are some simple CGI program in the "cgi-bin/",you can copy them into the root document,by type "sudo cp -r cgi-bin /var/tina/www".
    6. Type "sudo update-rc.d tinad defaults" to make tina to be a serice.Default tina service is not start.
  5. Start
    • To start tina,you just need type "sudo service tinad start".if no error,it's all ok.
  6. Test
    • At this point the server should run and serve documents. If not, check the log file in the root document "/var/tina/www/log/tina.log".

I realize this isn't exactly comprehensive documentation.

Bugs

  • Crash when F5 is pressed for a comment.
    In chrome or other explorer,if F5 is pressed for a comment to refresh the page,then the Tina server may crash.
  • ERR_EMPTY_RESPONSE when F5 is pressed for a comment.
    In chrome. if F5 is pressed for a comment to refresh the page,then the explorer may get a ERR_EMPTY_RESPONSE error.
  • Encoding error for request url.
    In the request url,if there are "%xx" string,then Tina will decode it.so if you have a file named "file%20.txt",then the url "xxx.xxx.xxx.xxx/file%20.txt" will get a 404(Not Found) response,because the "%20" will be decode into a "space".
